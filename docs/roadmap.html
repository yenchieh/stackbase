Findings carried into later tasks

U2 · Pinned prod deploys + guard rails #

Shipped

Status: Done.

Makefile prod-build/prod-push (api+frontend+backup, :latest+:<git-sha> → $(REGISTRY)); deploy renders overlays/prod, refuses on placeholder hits (change-me|replace-me|TODO|<[A-Z_]+>), then applies at :$(TAG). _guard-local-context prereq on apply verifies the kube API server is this machine/a private address (escape: GUARD_OK=1). PROD_KUBECONFIG ?= knob.

Post-review fix

TAG defaults to latest so a standalone make deploy ships the always-pullable tag (no more pinning to an unpushed/tarball SHA); prod-deploy = build+push then deploy TAG=$(SHA) for an immutable, rollback-able release.

Owner decision (2026-07-08): keep prod-build/prod-push — they're the manual pre-CI build/push path (isolated in prod-deploy); Phase-2 CI (Task 22) just automates them, it doesn't replace them.
